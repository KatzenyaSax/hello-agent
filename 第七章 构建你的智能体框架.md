## 本章目标

本章目标为，我们需要手写以下架构：

```
HelloAgent/
   │
   ├── core/                     # 核心框架层
   │   ├── agent.py              # Agent基类
   │   ├── llm.py                # HelloAgentsLLM统一接口
   │   ├── message.py            # 消息系统
   │   ├── config.py             # 配置管理
   │   └── exceptions.py         # 异常体系
   │
   ├── agent/                    # Agent实现层
   │   ├── simple_agent.py       # SimpleAgent实现
   │   ├── react_agent.py        # ReActAgent实现
   │   ├── reflection_agent.py   # ReflectionAgent实现
   │   └── plan_solve_agent.py   # PlanAndSolveAgent实现
   │
   ├── tool/                     # 工具系统层
       ├── base.py               # 工具基类
       ├── registry.py           # 工具注册机制
       ├── chain.py              # 工具链管理系统
       ├── async_executor.py     # 异步工具执行器
       └── builtin/              # 内置工具集
           ├── calculator.py     # 计算工具
           └── search.py         # 搜索工具
```

在里面有眼熟的，agent/ 目录下面就有第四章中手写过的 ReAct、Plan-and-Resolve 和 Reflection 三种模式的 Agent；我们的 llm 接口则移到 core 了。

另外，第四章的 ReAct 模式中已经有过工具构建的经验，将在 tool/ 目录下面进行定义，包括工具执行、注册发现、工具本体等。


## 体验现有的 Hello Agent 框架

官方提供了已经实现好的代码，使用 pip 可以进行安装：

```bash
pip install "hello-agents==0.1.1"
```

之后我们定义一个 testHelloAgent 来学习一下如何构建这个 Agent，我们来阅读一下。

### 构建 Agent

```py
# 创建LLM实例 - 框架自动检测provider
llm = HelloAgentsLLM()

# 或手动指定provider（可选）
# llm = HelloAgentsLLM(provider="modelscope")

# 创建SimpleAgent
agent = SimpleAgent(
    name="AI助手",
    llm=llm,
    system_prompt="你是一个有用的AI助手"
)
```

构建一个简单的 SimpleAgent ，与我们第四章构建 Agent 的方法几乎完全一样：我们将 LLM 调用接口 `llm` 作为变量传入 Agent。

### 简单对话

```py
# 基础对话
response = agent.run("你好！请介绍一下自己")
print(response)
```

我们看到，`run` 函数同样时作为 Agent 暴露给外界的运行方式。

### Function Calling

```py
 添加工具功能（可选）
from hello_agents.tools import CalculatorTool
calculator = CalculatorTool()
# 需要实现7.4.1的MySimpleAgent进行调用，后续章节会支持此类调用方式
# agent.add_tool(calculator)

# 现在可以使用工具了
response = agent.run("请帮我计算 2 + 3 * 4")
print(response)
```

这一步类似第四章中的构建：将方法作为 callable，注册到 Agent，然后 agent 才能激活工具意图。

## HelloAgentLLM 拓展

我们根据第四章的 HelloAgentLLM 进行拓展，将这个原本只能进行最基本的发送 LLM 请求与处理流式响应的基础 Cli，重构为更加完善的基础 Cli。

官方的说法是， HelloAgentLLM 原本只支持 OpenAI 规范的模型，所以我们将其需要改造成支持更多供应商，并实现根据模型的不同自动化适配规范和协议。


### 继承 HelloAgentLLM 的 llm.py

core/ 内构建 llm.py，我们已经实现了 HelloAgentLLM 类，我们直接让 llm 继承已有库就行。另外，已有库不建议进行修改，进行继承是一种较为标准的做法。

```py
# 继承 HelloAgentLLM
import HelloAgentLLM import HelloAgentLLM
class llm(HelloAgentLLM):
```
