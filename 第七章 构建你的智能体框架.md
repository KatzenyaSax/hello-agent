```

   │   ├── message.py            # 消息系统
   │   ├── config.py             # 配置管理
   │   └── exceptions.py         # 异常体系
   │

   │
```

### 标准化消息格式：message

Message 类是为了让上下文的格式进行统一，其继承了 `BaseModel` 类，这是 python 中专门用于定义数据类的库，只要继承就可以实现方便的数据管理。

单个 Massage 类对象的信息以 kv 对形式存储，分别为初始化时传入的参数：`role: MessageRole` 和 `content: str`。初始化：

```py
    def __init__(self, content: str, role: MessageRole, **kwargs):
        super().__init__(
            content=content,
            role=role,
            timestamp=kwargs.get('timestamp', datetime.now()),
            metadata=kwargs.get('metadata', {})
        )
```

role 的取值被固定为四个字符串，这是为了直接对应 OpenAI API 的规范，openAI 规范中，LLM 以下四个字符串来判断该信息的来源和作用:

```py
MessageRole = Literal["user", "assistant", "system", "tool"]
```

`to_dict` 函数则将 Message 对象转成 openAI 支持的规范字典格式。

```py
    def to_dict(self) -> Dict[str, Any]:
        """转换为字典格式（OpenAI API格式）"""
        return {
            "role": self.role,
            "content": self.content
        }
```

`_str_` 函数就是直接将自带格式转成字符串的：

```py
    def __str__(self) -> str:
        return f"[{self.role}] {self.content}"
```

### 对外暴露接口：core/agent

我们创建一个 Agent 接口，来规定了一个 agent 需要实现哪些方法、应当拥有哪些行为，通过继承 ABC(Abstract Class) 可以实现类的接口化：不能创建实例、方法不能实现。

```py
class Agent(ABC):
    """Agent基类"""
```

首先，我们看看其初始化的输入：最重要的是 ``llm: LLM`、`system_prompt: Optional[str]` 和 `config: Optional[Config]` 三个。分别为 openAI 规范的大模型接口，必须传入、字符串形式的系统提示词、自行定义的 Config 类。

另外，如果没有输入 Config 实例，需要创建一个默认配置。

`_histor` 为 `Message` 列表，用于存储我们自定义规范化的上下文。

```py
    def __init__(
        self,
        name: str,
        llm: LLM,
        system_prompt: Optional[str] = None,
        config: Optional[Config] = None
    ):
        self.name = name
        self.llm = llm
        self.system_prompt = system_prompt
        self.config = config or Config()
        self._history: list[Message] = []
```

实现该抽象类的 Agent 类必须包含以下方法：

```py
    @abstractmethod
    def run(self, input_text: str, **kwargs) -> str:
        """运行Agent"""
        pass
    
    def add_message(self, message: Message):
        """添加消息到历史记录"""
        self._history.append(message)
    
    def clear_history(self):
        """清空历史记录"""
        self._history.clear()
    
    def get_history(self) -> list[Message]:
        """获取历史记录"""
        return self._history.copy()
    
    def __str__(self) -> str:
        return f"Agent(name={self.name}, provider={self.llm.provider})"
```

首先通过注解 `@abstractmethod` 规定实现类必须实现 `run` 函数，用以调用大模型。

三个管理上下文的函数：`add_message` `clear_history` `get_history`。






## tool：工具

Agent 必须能够调用工具，官方的内容编排我实在是无力吐槽，过完 core 直接上手 agent，还一上来就要继承官方给的包，那这跟手写还有什么关系？我直接用你的包不就完了，为啥还要自己实现？

总之，我们先在 tool 包内搞定工具。

### tool

我们最先定义一个抽象类 Tool，为工具定义规范：

```py
from abc import ABC, abstractmethod
from typing import List, Dict, Any
from .toolParameter import ToolParameter

class Tool(ABC):
    """工具基类"""

    def __init__(self, name: str, description: str):
        self.name = name
        self.description = description

    @abstractmethod
    def run(self, parameters: Dict[str, Any]) -> str:
        """执行工具"""
        pass

    @abstractmethod
    def get_parameters(self) -> List[ToolParameter]:
        """获取工具参数定义"""
        pass

    def to_openai_schema(self) -> Dict[str, Any]:
        """转换为 OpenAI function calling schema 格式

        用于 FunctionCallAgent，使工具能够被 OpenAI 原生 function calling 使用

        Returns:
            符合 OpenAI function calling 标准的 schema
        """
        parameters = self.get_parameters()

        # 构建 properties
        properties = {}
        required = []

        for param in parameters:
            # 基础属性定义
            prop = {
                "type": param.type,
                "description": param.description
            }

            # 如果有默认值，添加到描述中（OpenAI schema 不支持 default 字段）
            if param.default is not None:
                prop["description"] = f"{param.description} (默认: {param.default})"

            # 如果是数组类型，添加 items 定义
            if param.type == "array":
                prop["items"] = {"type": "string"}  # 默认字符串数组

            properties[param.name] = prop

            # 收集必需参数
            if param.required:
                required.append(param.name)

        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": {
                    "type": "object",
                    "properties": properties,
                    "required": required
                }
            }
        }
```

初始化中，我们输入了 `name` 和 `description`，用来告诉 LLM 这是什么工具、什么时候需要调用这个工具。

在 `run` 函数中，需要实现工具逻辑，同时将返回值封装为 `Dict[str, Any]` 的形式。

`get_parameters` 函数中，我们要向外界暴露出该工具的 `name` 和 `description`，便于进行工具的注册发现。里面的 `ToolParameter` 是一个数据类，用来描述工具更加详细的信息，需要我们自行定义。

最后，我们使用 `to_openai_schema` 函数，将工具转化为 openAI 格式的 `json schema`，shcema的格式为：

```JSON
{
    "type": "function",
    "function": {
        "name": self.name,
        "description": self.description,
        "parameters": {
            "type": "object",
            "properties": properties,
            "required": required
        }
    }
}
```

### ToolParameter

ToolParameter 是一个数据类，需要继承 BaseModel，其内包含的参数有：

```py
    """工具参数定义"""
    name: str
    type: str
    description: str
    required: bool = True
    default: Any = None
```

### ToolRegistry

有了具体的工具，接下来就要进行注册发现，我们定义一个 `ToolRegistry` 类作为工具注册表。

类内最重要的成员变量是 `_tools` 和 `_functions`，分别用来存储 `完整的 Tool 对象` 和 `函数 + 描述`，因为我们**支持两种注册方式**：Tool & Callable。

```py
        self._tools: dict[str, Tool] = {}
        self._functions: dict[str, dict[str, Any]] = {}
```

#### 工具注册

Tool 的注册很简单，直接传入一个 Tool 就行：

```py
    def register_tool(self, tool: Tool):
        """注册Tool对象"""
        if tool.name in self._tools:
            print(f"⚠️ 警告:工具 '{tool.name}' 已存在，将被覆盖。")
        self._tools[tool.name] = tool
        print(f"✅ 工具 '{tool.name}' 已注册。")
```

`register_function` 函数为 Callable 的注册，传入参数的 `func: Callable[[str], str]` 表示的是输入为字符串，返回值也为字符串的函数引用。

```py
    def register_function(self, name: str, description: str, func: Callable[[str], str]):
        """
        直接注册函数作为工具（简便方式）

        Args:
            name: 工具名称
            description: 工具描述
            func: 工具函数，接受字符串参数，返回字符串结果
        """
        if name in self._functions:
            print(f"⚠️ 警告:工具 '{name}' 已存在，将被覆盖。")

        self._functions[name] = {
            "description": description,
            "func": func
        }
        print(f"✅ 工具 '{name}' 已注册。")
```

`get_tools_description` 函数则能够向外界暴露注册表内包含的工具。


### 自定义 Callable 工具

我们自定义一个数学表达式求值函数工具 tool\builtin\calculator.py。

如果我们想要执行函数工具，或是想要获取这个函数的引用，我们就必须在 ToolRegister 里追加函数。

#### 函数工具的执行 & 获取

在 ToolRegister 内追加函数：

```py
    def execute_function(self, name: str, *args, **kwargs) -> Any:
        """执行注册的函数工具"""
        if name not in self._functions:
            raise ValueError(f"工具 '{name}' 未注册。")
        func = self._functions[name]["func"]
        return func(*args, **kwargs)
    
    def get_function(self, name: str) -> Callable[[str], str]:
        """获取注册的函数工具"""
        if name not in self._functions:
            raise ValueError(f"工具 '{name}' 未注册。")
        return self._functions[name]["func"]
```

### 自定义 Tool 工具

我们自定义一个 search.SearchTool 类，继承 Tool 类：

```py
class SearchTool(Tool):
    """
    智能混合搜索工具，使用 SerpApi 和 Tavily API 进行网页搜索。
    """

    def __init__(self):
        super().__init__(
            name="search",
            description="一个智能网页搜索引擎。支持混合搜索模式，自动选择最佳搜索源。"
        )
        self.tavily_key = os.getenv("TAVILY_API_KEY")
        self.serpapi_key = os.getenv("SERPAPI_API_KEY")
        self.available_backends = []
        self._setup_backends()
        print(f"SearchTool initialized with backend: {self.available_backends}")

```

#### 选择搜索引擎

Hello Agent 到现在，已经展示了两种搜索引擎：tavily 和 serpapi，SearchTool 中我们想要让动态选择这两种搜索引擎，所以我们有 `_setup_backends` 函数：

```py
    def _setup_backends(self):
        """
        设置可用的搜索后端。
        """
        if self.tavily_key:
            self.available_backends.append("tavily")
        elif self.serpapi_key:
            self.available_backends.append("serpapi")
        if not self.available_backends:
            raise ValueError("No available search backends. Please set TAVILY_API_KEY or SERPAPI_API_KEY.")
```

根据环境变量中是否有 Key 来选择，优先选择 tavily。

#### tavily 搜索

在 hello agent 之前的章节中，已经展示过如何使用 tavily api 了，这里定义函数 `_search_tavily`：

```py
    def _search_tavily(self, query: str) -> Dict[str, Any]:
        """使用Tavily搜索"""
        self.tavily_client = TavilyClient(api_key=self.tavily_key)

        response = self.tavily_client.search(query=query, max_results=3)

        if response.get('answer'):
            result = f"💡 AI直接答案:{response['answer']}\n\n"
        else:
            result = ""

        result += "🔗 相关结果:\n"
        for i, item in enumerate(response.get('results', [])[:3], 1):
            result += f"[{i}] {item.get('title', '')}\n"
            result += f"    {item.get('content', '')[:150]}...\n\n"

        return result
```

先创建客户端，再向 tavily 服务器发送请求，然后处理 response；max_results 表示返回几条结果。

```py
load_dotenv()

class LLM:
    def __init__(self, model: str = None, apiKey: str = None, baseUrl: str = None, timeout: int = None):
        # 1.初始化客户端。优先使用传入参数，如果未提供，则从环境变量加载。
        self.model = model or os.getenv("LLM_MODEL_ID")
        apiKey = apiKey or os.getenv("LLM_API_KEY")
        baseUrl = baseUrl or os.getenv("LLM_BASE_URL")
        timeout = timeout or int(os.getenv("LLM_TIMEOUT", 60))
        if not all([self.model, apiKey, baseUrl]):
            raise ValueError("模型ID、API密钥和服务地址必须被提供或在.env文件中定义。")

        # 2.构建一个 OpenAI 规范的 LLM 调用接口
        self.client = OpenAI(api_key=apiKey, base_url=baseUrl, timeout=timeout)

    def think(self, messages: List[Dict[str, str]], temperature: float = 0) -> str:
        """
        调用大语言模型进行思考，并返回其响应。
        """
        print(f"🧠 正在调用 {self.model} 模型...")
        try:
            # 通过LLM接口调用大模型，response为流式响应(stream=True)
            response = self.client.chat.completions.create(
                model=self.model,
                messages=messages,
                temperature=temperature,
                stream=True,
            )
            
            # 处理流式响应
            print("✅ 大语言模型响应成功:")
            collected_content = []
            for chunk in response:
                if not chunk.choices:
                    continue
                content = chunk.choices[0].delta.content or ""
                # print(content, end="", flush=True)
                collected_content.append(content)
            print()  # 在流式输出结束后换行
            return "".join(collected_content)

        except Exception as e:
            print(f"❌ 调用LLM API时发生错误: {e}")
            return None
```

```py
from abc import ABC, abstractmethod
from typing import List, Dict, Any
from .toolParameter import ToolParameter

class Tool(ABC):
    """工具基类"""

    def __init__(self, name: str, description: str):
        self.name = name
        self.description = description

    @abstractmethod
    def run(self, parameters: Dict[str, Any]) -> str:
        """执行工具"""
        pass

    @abstractmethod
    def get_parameters(self) -> List[ToolParameter]:
        """获取工具参数定义"""
        pass

    def to_openai_schema(self) -> Dict[str, Any]:
        """转换为 OpenAI function calling schema 格式

        用于 FunctionCallAgent，使工具能够被 OpenAI 原生 function calling 使用

        Returns:
            符合 OpenAI function calling 标准的 schema
        """
        parameters = self.get_parameters()

        # 构建 properties
        properties = {}
        required = []

        for param in parameters:
            # 基础属性定义
            prop = {
                "type": param.type,
                "description": param.description
            }

            # 如果有默认值，添加到描述中（OpenAI schema 不支持 default 字段）
            if param.default is not None:
                prop["description"] = f"{param.description} (默认: {param.default})"

            # 如果是数组类型，添加 items 定义
            if param.type == "array":
                prop["items"] = {"type": "string"}  # 默认字符串数组

            properties[param.name] = prop

            # 收集必需参数
            if param.required:
                required.append(param.name)

        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": {
                    "type": "object",
                    "properties": properties,
                    "required": required
                }
            }
        }
```

```py
from typing import Callable, Any, Dict
from .tool import Tool

class ToolRegistry:
    """HelloAgents工具注册表"""

    def __init__(self):
        self._tools: dict[str, Tool] = {}
        self._functions: dict[str, dict[str, Any]] = {}

    def register_tool(self, tool: Tool):
        """注册Tool对象"""
        if tool.name in self._tools:
            print(f"⚠️ 警告:工具 '{tool.name}' 已存在，将被覆盖。")
        self._tools[tool.name] = tool
        print(f"✅ 工具 '{tool.name}' 已注册。")
        
    def register_function(self, name: str, description: str, func: Callable[[str], str]):
        """
        直接注册函数作为工具（简便方式）

        Args:
            name: 工具名称
            description: 工具描述
            func: 工具函数，接受字符串参数，返回字符串结果
        """
        if name in self._functions:
            print(f"⚠️ 警告:工具 '{name}' 已存在，将被覆盖。")

        self._functions[name] = {
            "description": description,
            "func": func
        }
        print(f"✅ 工具 '{name}' 已注册。")

    def execute_function(self, name: str, *args, **kwargs) -> Any:
        """执行注册的函数工具"""
        if name not in self._functions:
            raise ValueError(f"工具 '{name}' 未注册。")
        func = self._functions[name]["func"]
        return func(*args, **kwargs)
    
    def get_function(self, name: str) -> Callable[[str], str]:
        """获取注册的函数工具"""
        if name not in self._functions:
            raise ValueError(f"工具 '{name}' 未注册。")
        return self._functions[name]["func"]
    
    def execute_tool(self, name: str, *args, **kwargs) -> Any:
        """执行注册的Tool对象"""
        if name not in self._tools:
            raise ValueError(f"工具 '{name}' 未注册。")
        tool = self._tools[name]
        return tool.run(*args, **kwargs)
    
    def get_tool(self, name: str) -> Tool:
        """获取注册的Tool对象"""
        if name not in self._tools:
            raise ValueError(f"工具 '{name}' 未注册。")
        return self._tools[name]

    def get_tools_description(self) -> str:
        """获取所有可用工具的格式化描述字符串"""
        descriptions = []

        # Tool对象描述
        for tool in self._tools.values():
            descriptions.append(f"- {tool.name}: {tool.description}")

        # 函数工具描述
        for name, info in self._functions.items():
            descriptions.append(f"- {name}: {info['description']}")

        return "\n".join(descriptions) if descriptions else "暂无可用工具"
```

```py
# my_calculator_tool.py
import ast
import operator
import math
def calculator(expression: str) -> str:
    """简单的数学计算函数"""
    if not expression.strip():
        return "计算表达式不能为空"

    # 支持的基本运算
    operators = {
        ast.Add: operator.add,      # +
        ast.Sub: operator.sub,      # -
        ast.Mult: operator.mul,     # *
        ast.Div: operator.truediv,  # /
    }

    # 支持的基本函数
    functions = {
        'sqrt': math.sqrt,
        'pi': math.pi,
    }

    try:
        node = ast.parse(expression, mode='eval')
        result = _eval_node(node.body, operators, functions)
        return str(result)
    except:
        return "计算失败，请检查表达式格式"

def _eval_node(node, operators, functions):
    """简化的表达式求值"""
    if isinstance(node, ast.Constant):
        return node.value
    elif isinstance(node, ast.BinOp):
        left = _eval_node(node.left, operators, functions)
        right = _eval_node(node.right, operators, functions)
        op = operators.get(type(node.op))
        return op(left, right)
    elif isinstance(node, ast.Call):
        func_name = node.func.id
        if func_name in functions:
            args = [_eval_node(arg, operators, functions) for arg in node.args]
            return functions[func_name](*args)
    elif isinstance(node, ast.Name):
        if node.id in functions:
            return functions[node.id]
```
